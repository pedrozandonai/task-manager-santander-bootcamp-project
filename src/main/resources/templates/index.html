Projeto Santander Boot Camp - Sistema de Gerenciamento de Tarefas Pessoais

Descrição do Projeto

Desenvolver um sistema de gerenciamento de tarefas pessoais em Java que permita aos usuários criar, organizar e monitorar suas tarefas diárias, semanais e mensais.

Objetivo do Projeto

O objetivo deste projeto é criar um sistema de gerenciamento de tarefas pessoais que ajude os usuários a manterem-se organizados e produtivos em suas atividades diárias.

Funcionalidades Principais

  • Cadastro de Tarefas: Os usuários podem criar tarefas, fornecendo informações como título, descrição, data de vencimento, prioridade, categoria, etc.
  • Organização de Tarefas: O sistema permite que os usuários organizem suas tarefas em listas ou categorias personalizadas para facilitar a gestão.
  • Lembretes e Notificações: Os usuários recebem lembretes e notificações sobre tarefas pendentes ou próximas do prazo de vencimento.
  • Visualização de Tarefas: Os usuários podem visualizar suas tarefas em diferentes modos, como lista, calendário ou kanban, dependendo de suas preferências.
  • Marcação de Tarefas Concluídas: Os usuários podem marcar tarefas como concluídas e arquivá-las para referência futura.

Design Patterns a serem Utilizados

  • Observer Pattern: Implemente o padrão Observer para notificar os usuários sobre lembretes e atualizações de tarefas.
  • Factory Method Pattern: Use o padrão Factory Method para criar objetos de tarefa e categoria de forma flexível e encapsulada.
  • Strategy Pattern: Utilize o padrão Strategy para implementar diferentes estratégias de notificação (por exemplo, notificações por e-mail, notificações por push, etc.).
  • Decorator Pattern: O padrão Decorator pode ser usado para adicionar funcionalidades adicionais às tarefas, como anexar arquivos, adicionar comentários, etc.
  • Singleton Pattern: Use o padrão Singleton para garantir que exista apenas uma instância do sistema de gerenciamento de tarefas em toda a aplicação.
